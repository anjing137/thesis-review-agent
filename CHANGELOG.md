# Changelog

本项目遵循语义化版本号。

## [0.6.0] - 2026-06-08

### Added

- 新增 `S/A/P/R` 稳定证据编号和证据核验索引。
- 新增实证研究论文与学理论文的差异化维度及权重。
- 新增结构化 `review.json` 校验，检查分数、维度、证据和修改建议。
- 新增 Python 自动加权、硬规则否决及评分明细输出。
- 新增正式双层评审报告：正文用于教师反馈，附录用于证据复核。
- 新增 `criteria.yaml` 单一规则源、自定义规则文件和 `--version`。
- 新增 Codex Skill 界面元数据和核心回归测试。

### Changed

- 评价体系由固定6维度升级为按论文类型生效的9维度体系。
- 参考文献数量规范统一为不少于18篇。
- 修改建议改为“修改位置、具体措施、修改目的”三要素。
- 报告正文改用正式、审慎的学院评审语体。
- 推荐工作流改为 `--prompt` 生成证据，再用 `--report-json` 生成报告。

### Fixed

- 修复将评价标准中的方法关键词误判为论文实际方法的问题。
- 修复稳健性内容存在于正文但未被识别的问题。
- 修复 `DD1`、自定义输出目录失效和小数分数等级边界等问题。
- 消除15篇与18篇参考文献阈值冲突。

### Removed

- 删除旧版未接入主流程的分析器、报告模板和重复规则实现。
- 不再在正式报告正文中显示内部技术编号。

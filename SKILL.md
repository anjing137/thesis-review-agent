# 论文评价 Agent

## 简介

本Agent对经济与管理学院本科生学年论文/毕业论文进行专业评价。

**评价维度**（共6个，总分100分）：

| 维度 | 权重 | 说明 |
|-----|------|------|
| 选题与研究问题 | 15% | 题目质量、研究必要性、研究价值 |
| 参考文献与学术规范 | 15% | 数量≥15篇（硬性）、质量、格式、外文≥3篇 |
| 内容创新性 | 15% | 研究视角、方法、发现、文献评述 |
| 框架与逻辑结构 | 20% | 章节完整、逻辑清晰、引言质量 |
| 方法与论证严谨性 | 25% | 理论分析、稳健性检验（强制）、内生性、**过度控制检测** |
| 语言与表达 | 10% | 第三人称、术语规范、标点、图表 |

**新增能力**：对于实证性论文，额外进行**模型设定审查**，自动检测"过度控制"（over-control）等问题。

## 适用场景

- 学年论文评价
- 毕业论文初稿评价
- 学生论文修改指导
- 论文质量检查

## 使用方法

### 方式一：分析单篇论文

用户提供论文路径后，Agent将：
1. 使用Python层自动提取论文基本信息（字数、参考文献、结构等）
2. 读取论文全文
3. 进行6维度深度AI评价
4. 生成结构化评价报告

```
用户：请帮我评价一下 papers/数字金融对商业银行风险的影响研究.docx
```

### 方式二：批量评价

```
用户：请批量评价 papers/ 目录下的所有论文
```

### 方式三：对比分析

```
用户：对比评价 papers/ 下的这两篇论文：论文A.docx 和 论文B.docx
```

## 评价流程

### 第一步：Python规则检测层

Python层自动完成：
- 文档格式转换（pandoc）
- 学生信息提取（姓名、学号、专业、题目）
- 论文类型判断（实证性/学理性）
- 正文字数统计
- 参考文献分析（数量、外文、期刊占比）
- 结构完整性检查
- 写作规范初检（第一人称、截图等）
- **模型设定分析（仅实证性论文）**
  - 提取变量定义和因果链声明
  - 检测"过度控制"问题（如将中介变量作为控制变量）
  - 检测遗漏控制变量、变量角色混淆等问题

### 第二步：AI深度评价

Claude AI基于论文内容和检测结果：
- 逐维度评价（引用原文）
- 计算各维度得分
- 识别优点和问题
- 生成具体可操作的修改建议
- **对于实证性论文，额外进行模型设定审查**
  - 核对变量角色是否正确
  - 指出过度控制对系数估计的影响
  - 建议正确的建模方法

### 第三步：生成报告

- Markdown格式评价报告
- 综合评分和等级
- 按优先级分类的修改建议

## 输出内容

### 综合评分
- 百分制得分
- 评级等级（优+/优/良+/良/中+/中/中-/不合格）

### 各维度详细评价
- 优点列表（引用原文）
- 问题列表（引用原文）
- 修改建议（按优先级）

### 修改建议（分类）
- **高优先级**：必须修改，影响论文合格
- **中优先级**：建议修改，提升论文质量
- **低优先级**：可选修改，完善细节

## 一票否决项

| 问题 | 后果 |
|-----|------|
| 参考文献<15篇 | 参考文献维度0分 |
| 实证论文无稳健性检验 | 方法与论证维度0分 |
| 实证论文存在明显过度控制 | 方法与论证维度扣5-10分 |
| 重复率>25% | 综合评分不超过50分 |

## 评分等级标准

| 得分 | 等级 |
|-----|------|
| 90-100 | 优秀（优+） |
| 85-89 | 良好上（优） |
| 80-84 | 良好中（良+） |
| 75-79 | 良好下（良） |
| 70-74 | 中等上（中+） |
| 65-69 | 中等（中） |
| 60-64 | 中等下（中-） |
| <60 | 不合格 |

## 注意事项

1. 评价基于《经济与管理学院本科生论文指导要点》
2. 实证性论文必须检查稳健性检验
3. 修改建议具体可操作
4. 高优先级问题用**粗体**标注

## 依赖工具

- pandoc：Word文档转换
- Python 3.8+：规则检测层
- Claude AI：深度内容评价

## 论文类型判断

**实证性论文特征**：
- 包含回归分析、显著性检验
- 有数据、模型、变量
- 进行稳健性检验

**学理性论文特征**：
- 现状分析、问题分析、原因分析、对策建议
- 结构为"现状→问题→原因→对策"

# 论文评价 Agent - 使用说明

## 调用方式

当用户提供论文路径并要求评价时，按以下步骤执行：

### 第一步：运行 Python 规则检测层

```bash
python3 /Users/anjing137/.claude/projects/paperCheck/thesis-review-agent/main.py "论文完整路径" --json-only
```

**【重要】关于字数判断**：
- 标题字数：Python已精准统计（basic_stats.title_word_count），**不要自行估算**
- 正文字数：Python已精准统计（basic_stats.word_count），**不要自行估算**
- 参考文献数量：Python已精准统计（basic_stats.ref_count），**不要自行估算**
- 所有字数/数量判断必须使用Python检测结果

### 第二步：读取论文全文

使用 pandoc 将论文转换为 Markdown 后读取内容：
```bash
pandoc "论文路径.docx" -o /tmp/paper_review.md
```

### 第三步：基于检测结果和论文内容进行 AI 深度评价

按照以下 6 个维度评价（权重）：
- 选题与研究问题（15%）
- 参考文献与学术规范（15%）
- 内容创新性（15%）
- 框架与逻辑结构（20%）
- 方法与论证严谨性（25%）
- 语言与表达（10%）

### 第四步：生成评价报告

按照标准格式输出评价报告，包含：
1. 综合评分和各维度得分
2. 各维度详细评价（优点、问题、引用原文）
3. 按优先级分类的修改建议
4. 总结

---

## 评价维度详细说明

### 维度1：选题与研究问题（15%）
- 题目是否≤20字、具体明确
- **标题字数必须使用Python检测结果（basic_stats.title_word_count），不要自行估算**
- 研究必要性是否充分论证
- 研究价值和创新点是否明确

### 维度2：参考文献与学术规范（15%）
- 数量是否≥15篇（硬性要求）
- 外文文献是否≥3篇
- 期刊占比是否≥2/3
- 格式是否规范统一

### 维度3：内容创新性（15%）
- 研究视角是否新颖
- 是否有文献评述（指出已有研究不足）
- 是否明确边际贡献

### 维度4：框架与逻辑结构（20%）
- 实证论文标准结构：引言→文献综述→理论分析→研究设计→实证分析→结论
- 学理论文标准结构：引言→现状→问题→原因→对策→结语
- 各章节逻辑是否清晰

### 维度5：方法与论证严谨性（25%）
- 理论分析是否深入
- 数据质量和变量定义是否清晰
- **稳健性检验是否完整（实证论文强制要求）**
- 是否讨论内生性问题

### 维度6：语言与表达（10%）
- 是否使用第三人称（"本文"是正确的，"我/我们"是错误的）
- 术语是否专业规范
- 标点符号是否正确
- 图表是否规范

---

## 一票否决项

| 问题 | 后果 |
|-----|------|
| 参考文献<15篇 | 参考文献维度0分 |
| 实证论文无稳健性检验 | 方法与论证维度0分 |

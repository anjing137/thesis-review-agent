# 论文评价工作流

## 推荐流程

### 一步完成分析+骨架生成

```bash
python3 main.py 论文路径.docx --report -o ./输出目录
```

输出：
- `论文名.md` - 论文markdown全文
- `论文名_评价报告_骨架.md` - AI填充用报告骨架

### AI评价

Agent读取骨架和论文markdown，填充AI评价内容，生成完整报告。

---

## 详细流程（可选）

### 第一步：Python规则检测

```bash
python3 main.py 论文路径.docx --report -o ./reviews
```

Python层自动完成：
- 文档格式转换（pandoc）
- 学生信息提取（姓名、学号、专业、题目）
- 论文类型判断（实证性/学理性）
- 参考文献分析（数量、外文、期刊占比）
- 结构完整性检查
- 写作规范初检（第一人称、截图等）
- 模型设定分析（仅实证性论文）

### 第二步：AI深度评价

读取骨架文件 + 论文markdown，基于检测数据进行6维度评价。

## 字数统计（LLM辅助）

请阅读论文全文，判断摘要和正文范围，并在回复中包含以下标记：

```
【摘要】
[粘贴摘要完整内容，包括中英文摘要]
【摘要结束】

【正文内容】
[粘贴从引言开始到参考文献之前的所有内容]

```

**字数要求**：
- 摘要：300-500字
- 正文：≥8000字

## 评价维度（权重）

| 维度 | 权重 | 说明 |
|-----|------|------|
| 选题与研究问题 | 15% | 题目质量（≤20字）、研究必要性、研究价值 |
| 参考文献与学术规范 | 15% | 数量≥15篇、外文≥3篇、期刊占比 |
| 内容创新性 | 15% | 研究视角、边际贡献、文献评述 |
| 框架与逻辑结构 | 20% | 章节完整、逻辑清晰 |
| 方法与论证严谨性 | 25% | 理论分析、稳健性检验（实证论文强制）、内生性 |
| 语言与表达 | 10% | 第三人称、术语规范 |

## 一票否决项

| 问题 | 后果 |
|-----|------|
| 参考文献<15篇 | 参考文献维度0分 |
| 外文文献<3篇 | 外文文献不足，应指出 |
| 实证论文无稳健性检验 | 方法与论证维度0分 |

---

请生成完整的评价报告，包括：
1. 综合评分和各维度得分
2. 各维度详细评价（优点、问题）
3. 按优先级分类的修改建议
4. 总结

**注意**：
- 实证性论文**必须**评价稳健性检验是否完整
- 论文使用了"本文"等第三人称表达，这是正确的

---

## 模型设定评价模块（仅适用于实证性论文）

当论文为实证性论文时，请在原有6维度评价基础上，**额外**增加以下模型设定评价。

### 变量与因果链提取

请从论文中提取：
1. 变量定义表（被解释变量、核心解释变量、控制变量、中介变量）
2. 因果链声明（如：X通过M影响Y）
3. 检验是否存在过度控制问题

### 过度控制检测

如果论文声明了"X → M → Y"的因果链，但模型中控制了M，则存在过度控制。

**示例问题**：
- 变量`rd`（研发投入）在理论部分被声明为"耐心资本影响企业创新"的中介变量
- 但在主回归模型中，`rd`被作为控制变量
- 这会阻断"耐心资本→研发投入→企业创新"的中介路径

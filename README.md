# Thesis Review Agent

经济与管理学院本科生论文评价 Agent，基于《论文指导要点》进行 6 维度深度评价，生成结构化评价报告和修改建议。

## 功能特性

- **6 维度评价体系**：选题与研究问题、参考文献与学术规范、内容创新性、框架与逻辑结构、方法与论证严谨性、语言与表达
- **Python 规则检测层**：自动提取论文基本信息、字数统计、参考文献分析、结构完整性检查、写作规范检测
- **模型设定审查（实证论文专用）**：自动检测"过度控制"（over-control）问题，识别中介变量角色混淆
- **多种使用方式**：单篇评价、批量评价、对比评价
- **结构化报告输出**：Markdown 格式评价报告，含综合评分、等级、各维度详细评价、修改建议

## 评价维度与权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 方法与论证严谨性 | 25% | 理论分析、稳健性检验（实证论文强制）、内生性、过度控制检测 |
| 框架与逻辑结构 | 20% | 章节完整、逻辑清晰、引言质量 |
| 选题与研究问题 | 15% | 题目质量、研究必要性、研究价值 |
| 参考文献与学术规范 | 15% | 数量≥15篇（硬性）、外文≥3篇、期刊占比 |
| 内容创新性 | 15% | 研究视角、方法、发现、文献评述 |
| 语言与表达 | 10% | 第三人称、术语规范、标点、图表 |

## 评分等级

| 得分 | 等级 |
|------|------|
| 90-100 | 优秀（优+） |
| 85-89 | 良好上（优） |
| 80-84 | 良好中（良+） |
| 75-79 | 良好下（良） |
| 70-74 | 中等上（中+） |
| 65-69 | 中等（中） |
| 60-64 | 中等下（中-） |
| <60 | 不合格 |

## 一票否决项

| 问题 | 后果 |
|------|------|
| 参考文献 < 15 篇 | 参考文献维度 0 分 |
| 实证论文无稳健性检验 | 方法与论证维度 0 分 |
| 实证论文存在过度控制 | 方法与论证维度扣 5-10 分 |

## 使用方法

### 推荐：生成报告骨架（包含所有Python检测数据）

```bash
# 生成报告骨架 + 论文markdown
python3 main.py "论文路径.docx" --report -o ./reviews

# 生成报告骨架（输出到当前目录）
python3 main.py "论文路径.docx" --report
```

### 单独使用Python规则检测

```bash
# 仅输出分析摘要（不生成骨架）
python3 main.py "论文路径.docx" --json-only

# 保存JSON分析结果
python3 main.py "论文路径.docx" --output ./reviews
```

## 实证论文：模型设定审查

对于实证性论文，额外进行模型设定审查：

### 过度控制检测（Over-control Detection）

当中介变量被作为控制变量放入模型时，会阻断中介路径，导致直接效应估计偏误。

**示例**：

| 研究主题 | 中介变量 | 错误做法 | 正确做法 |
|---------|---------|---------|---------|
| 耐心资本 → 企业创新 | 研发投入 (rd) | 将 rd 作为控制变量 | 移除 rd，单独做中介效应分析 |
| 数字金融 → 企业创新 | 融资约束 | 控制融资约束 | 控制融资约束可能阻断中介路径 |

### 检测流程

1. 从论文中提取变量定义和因果链声明
2. 核对变量在因果链中的角色（被解释变量/核心解释变量/中介变量/控制变量）
3. 检测模型中是否将中介变量错误地作为控制变量
4. 报告发现的问题及修正建议
